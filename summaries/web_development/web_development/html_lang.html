Content categories

Every HTML can be categorized in one or more content categories. The categories describe share behavior and associated rules.

Elements exist that are not part of these categories.

  1. Main categories: describe common rules shared by many elements
  2. Form-related content categories
  3. Specific content categories: describes rare categories shared only by a few elements
Read more: Content categories (Mozilla)

Main Categories

  • Metadata content

    They modify the presentation or behavior of the rest of the document, set up links to other documents.

  • Flow content

    Contain text or embedded content.

    <p> <br> <a>

    <h1> <h2> <h3> <h4> <h5> <h6>

    <blockquote> <cite> <q>

    <img>

    <ul> <ol> <li>

    Atributes on Ordered Lists:

    • reversed: boolean atribute that puts list items in reverse order (numbered from high to low) reversed=true
    • start: an integer to start counting from the list items. For example, if we want to start numbering at 3 start="3"

  • Sectioning content

    Create sections that define the header, footer elements, and heading content.

  • Heading content

    Defines the title of a section.

    <h1> <h2> <h3> <h4> <h5> <h6>

    • Headings use size to indicate their relative importance, but we should avoid using heading tags to resize text
    • Avoid skipping heading levels
    • You should use only one <h1> per page. It's the most important heading and tells you the purpose of the page. You wouldn't have a book with more that one title!

  • Phrasing content

    Defines the text and the mark-up it contains.

    <br>

    • <br> are used to produce a line break, useful for poems or writing an address. However, creating paragraphs using <br> is a bad practice, it's frustrating and difficult to understand for people using screen readers.

    <cite>

    • <cite> is used to include a reference to the source of quoted material contaied with a <blockquote> or <q> element.

    <a>

    <img>

  • Embedded content

    Imports another resource or inserts content from another mark-up language into the document.

    <img>

    • Omiting the alt altogether indicates that the image is a key part of the content and there's no textual equivalent available
    • Setting the attribute to an empty string alt="" indicates that the image is not a key part of the content, it's a decoration, and non-visual browsers can omit it
    • The title atribute is not a good substitute for the alt attribute. It shouldn't be used as additional captioning information, the user may never see it if they don't hover over the image. If the image needs a caption, use the figure and figcaption elements
    • Also, avoid duplicating the alt attribute's text on the title atribute. Screen readers read the same description twice, which is confusing for users

  • Interactive content

    Includes elements that are specifically designed for user interaction.

    <a>

    • When using the target="_blank" atribute on <a>, add rel="noreferrer noopener" to avoid security issues
    • A common mistake is to only link the words "click here" or "here". It's a best practice to use descriptive text as the link. For example, "Learn more about our products". This greatly helps people using screen readers
    • You should only use a hyperlink for navigation to a real URL. If it's for something else, use <button>

  • Palpable content

    Content is palpable when it's neither empty or hidden; it's content that is rendered and substantive. Elements whose model is flow content or phrasing content should have at least one node which is palpable.

    <img>
