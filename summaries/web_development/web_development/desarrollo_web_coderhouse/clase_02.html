Listas

HTML permite agrupar elementos que tienen más significado en forma conjunta. Por ejemplo, el menú de navegación de un sitio web está formado por un grupo de palabras. Cada palabra tiene sentido por sí misma, de forma conjunta forman el menú de navegación de la página. Su significado conjunto es mayor que por separado. Esto se denomina como listas.

Algunos usos de las listas son:

  • Enumeración de pasos o secuencias. Enumeración de propiedades o características técnicas
  • Galerías de imágenes (en caso que todas las imágenes formen parte del mismo álbum y tengan la misma info - título, imagen, descripción-)
  • Botoneras (cada botón forma parte del listado de opciones que tenemos para navegar nuestra página)

Tipos de listas

  • Listas no ordenadas
  • Listas ordenadas
  • Listas de definición

Un <li> es el único hijo que pueden tener los <ol> y <ul>. Dentro de un <ol> o <ul> no se pueden poner imágenes, párrafos, etc. Adentro de los <li> sí se pueden poner otros elementos

Tablas

Conjunto de celdas organizadas dentro de las cuales podemos alojar distintos contenidos. Sirven para representar información tabulada, en filas y columnas.

  • Las tablas se definen por la etiqueta <table>
  • Se describen por filas de arriba a abajo y luego por columnas de izquierda a derecha
  • Las filas se definen con <tr>
  • Dentro de cada fila va a haber diferentes celdas. Las celdas se definen con <td>

Formularios

Contiene etiquetas para que el usuario ingrese información que luego va a ser enviada a un archivo encargado de procesarla.

Se usa la etiqueta <form> que requiere tres atributos para funcionar:

  • Action: indica cuál es el archivo que recibe y procesa los datos. Debe ser de un lenguaje del lado del servidor (PHP, ASP, JSP). Si no indicamos un valor, por defecto es el mismo archivo donde está el formulario

  • Method: indica la forma en la que se recopilan y envían los datos. Hay dos métodos de envío:

    • GET: la información viaja por la barra de direcciones a continuación del nombre del archivo
    • POST: la información viaja junto a los encabezados del HTML (será "invisible)"

    Si no indicamos un valor, el default es GET. Los valores de estos atributos se escriben en mayúscula.

  • Enctype: cómo se van a codificar los datos a enviar

Ingreso de texto

Existen tres controles generales para el ingreso de texto:

  • Cajas de texto de una sola línea
  • Cajas para el ingreso de contraseñas (el contenido no será visible)
  • Cajas para contenido multilínea

Input

Indica que es un campo donde el usuario va a escribir algo. La etiqueta no se cierra.

Requiere que con el atributo type se le diga el tipo que es (text, email, password, etc.)

Con el atributo name se indica cómo se llama la información que estamos enviando.

Si usamos type="submit" creamos un botón que permite enviar la información ingresada en el formulario.

El atributo placeholder permite escribir un texto que ocupa el lugar del texto que se va a escribir. Un ejemplo sería para indicar cómo escribir un número de teléfono: placeholder="11 0000 1111"

Controles de selección

Es para los casos en los que no queremos que el usuario ingrese libremente un texto, sino que le damos una serie de opciones definidas. En todos los casos, el dato que nos llega al elegir una opción se define desde el atributo value.

Hay tres grupos de controles de selección:

  • Botones de radio: solo se puede elegir una opción

  • Casillas de chequeo: de toda la lista de opciones, el usuario puede elegir una, todas o ninguna

  • Menú desplegable: (combo-box, selector o menú) solo se puede elegir una opción

Etiqueta Label

Define formalmente a cada elemento de un formulario, lo que ayuda a la accesibilidad.

El atributo principal es for que va referenciar a label con su elemento en el formulario. El valor del atributo for debe ser igual al valor del atributo id o name del elemento.

Conjunto de campo

La etiqueta fieldset tiene como objetivo crear grupos de elementos del formulario que tienen un mismo propósito. La etiqueta legend define formalmente el propósito del elemento fieldset.

Audio y video

Video

Al igual que con img, ccon video incluimos la ruta del archivo con el atributo src

Dentro de las etiquetas video incluimos un texto que se mostrará en caso de que el navegador no pueda mostrar el video.

Atributos de video

  • controls: basta con escribir el atributo solamente, sin valor. Si está presente, el navegador incluye controles de reproducción y volumen
  • autoplay: igual que con controls, basta con que el atributo esté presente para que se active la reproducción automática del video. Para remover la reproducción automática tenemos que borrar el atributo. Se recomienda no poner videos que se reproduzcan automáticamente para mejorar la experiencia de usuario
  • loop: si se incluye el atributo, el video se reproduce desde el inicio al finalizar
  • muted:atributo booleano que indica el setting por defecto del audio del video. Su valor por defecto es false (el audio se reproducirá con el video)
  • height y width: especificar el valor solo en pixels, no porcentajes

Audio

Comparte atributos con video: controls, autoplay y loop. Tienen el mismo funcionamiento.

El atributo preload se usa en el elemento audio para almacenar temporalmente (buffering) archivos de gran tamaño. Puede tener 3 valores:

  • none: no almacena temporalmente el archivo
  • auto: almacena temporalmente el archivo
  • metadata: almacena temporalmente solo los metadatos del archivo

Tanto en video como en audio se pueden especificar múltiples fuentes de archivos con formatos diferentes. Esto ayuda a que el navegador pueda cargar el video/audio en caso de no poder leer una extensión particular.

Imágenes

Para insertar una imagen basta con usar la etiqueta <img>, no es necesario cerrar la etiqueta como con video o audio.

  • También requiere del atributo source que le indique dónde está el archivo a mostrar
  • title: incluye un título que se va a mostrar cuando pasemos con el mouse por encima de la imagen. No es posible modificar la fuente o color del texto del title, ese formato lo determina cada navegador
  • alt: atributo muy importante para accesibilidad. Es un texto que debe representar de forma clara y concisa de qué se trata la imagen. No es necesario incluir un alt para imágenes decorativas que no agreguen al contenido de la página. Es una buena práctica incluir siempre un alt en las imágenes
  • Se pueden usar los atributos weight y height para determinar las dimensiones de la imagen

Enlaces

Los enlaces se usan para relacionar partes del mismo documento. Para crear un enlace es necesario usar la etiqueta <a> con el atributo href que va a establecer el destino al que apunta el enlace.

  • Enlaces relativos: apuntan a páginas dentro del mismo proyecto

  • Enlaces absolutos: apuntan a un documento fuera del proyecto, es necesario especificar la URL completa

  • Enlaces internos: permiten movernos a secciones dentro de la misma página utilizando un id. También podemos usar como destino una sección específica de una página distinta
