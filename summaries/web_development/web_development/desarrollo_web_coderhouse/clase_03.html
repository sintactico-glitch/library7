Reglas sintácticas

CSS (Cascading Style Sheets) es un lenguaje web para aplicar fomato visal (color, tamaño, separación y ubicación) al HTML.

Los cambios se aplican en cascada, es decir, la propiedad que definamos último va a tener prioridad frente a las definidas en la parte superior del documento.

  1. Cada declaración CSS está formada por un juego de pares (propiedad:valor;)

  2. No se ve afectado por los espacios en blanco. Las propiedades se pueden escribir de corrido o una debajo de la otra
  3. Siempre que la propiedad represente un número es necesario indicar en qué unidad de expresa (px, em, rem, %)

Padres e hijos

Insertar CSS

  1. Forma externa: dentro de la etiqueta <head> llamamos al archivo CSS que necesitemos

  2. Forma interna (por etiqueta): usando la etiqueta <style>, preferentemente en el <head>

  3. Forma interna (por atributo): muy poco recomendable, es muy difícil de mantener ya que tenemos el código CSS en cada etiqueta HTML. No permite reutilizar el CSS

Class & ID

Class

Son atributos que le pertenecen a HTML5 que vamos a usar para darle una estructura visual a nuestro sitio web. También permite identificar elementos.

Para aplicar una clase, en HTML usamos el atributo class y el valor es el nombre de la clase que vamos a usar en CSS.

Es recomendable que el nombre de la clase describa lo que va a hacer. En el ejemplo el nombre de la clase está describiendo lo que vamos a hacer después en CSS.

La clase permite llamar a todas las etiquetas que tengan esa clase. Distintos tipos de etiquetas pueden tener la misma clase, por ejemplo un <h2> y un <p>. Una etiqueta HTML puede tener más de una clase, esto no sucede con los ID.

Desde CSS, el nombre de la clase debe comenzar con letras pero podemos poner el nombre que querramos. Siempre se pone un punto (.) adelante de la clase al escribir en CSS.

ID

Se suele utilizar para nombrar porciones de código, sectores, secciones. Desde CSS podemos poner el nombre que queramos al ID pero debe comenzar con letras. Se pone un # adelante de los ID al escribir CSS.

En HTML usamos el atributo id y el valor va a ser la clase que definimos en CSS.

Cascada

La cascada es un algoritmo que define como combinar valores de propiedades de distintas fuentes. Es el núcleo de CSS.

Solo las declaraciones CSS (pares propiedad/valor) participan de la cascada.

Origen de las declaraciones CSS

  • Hoja de estilo del navegador(user-agent stylesheets): son las hojas de estilo que dan el estilo por defecto a los navegadores. Distintos navegadores tiene diferentes user-agent stylesheets.

    Con frecuencia se hace un reset de las propiedades de la user-agent stylesheet, haciendo que propiedades específicas tengan valores conocidos (ej: ponerle margin:0 al body)

  • Hoja de estilo del autor (author stylesheets): es el tipo más común de hoja de estilo, es la que usamos para personalizar el estilo de un sitio.

  • Hoja de estilo del usuario (user stylesheets): el usuario del sitio puede sobreescribir el estilo por default del navegador usando una hoja de estilo personalizada. Un uso posible sería aumentar el tamaño de fuente si tuviera problemas de visión.

Orden de la cascada

  1. Filtra todas las reglas de diferentes fuentes para mantener solo las reglas que aplican a cada elemento (las reglas cuyo selector y elemento coincidan y que estén en el @media apropiado)

  2. Después ordena estas reglas de acuerdo a su importancia (si tienen o no !important) y de acuerdo a su origen.

    La cascada va en orden ascendente, por lo que los valores !important de la hoja de estilo definida por el usuario tiene precedencia sobre los valores normales de la hoja de estilo del navegador

    Origen Importancia
    1 navegador normal
    2 usuario normal
    3 autor normal
    4 animaciones
    5 autor !important
    6 usuario !important
    7 navegador !important
    8 transiciones
  3. En caso de igualdad, lo que define cuál se aplica es la especificidad del valor

Animaciones CSS y la cascada

Las animaciones CSS (que usan @keyframes) definen animaciones entre estados. Los keyframes no forman parte de la cascada, lo que significa que CSS toma los valores de un solo @keyframes y nunca combina múltiples valores juntos.

Cuando varios keyframes son apropiados, elige el último definido en el documento más importante, nunca los va a combinar.
